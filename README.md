# Fully Decentralized Encrypted Peer-to-Peer Chat Engine

A proof-of-concept, serverless, end-to-end encrypted (E2EE) messaging application built entirely from scratch using Python's native `socket` and `threading` libraries. 

This project demonstrates how two distinct computer nodes can dynamically discover each other, negotiate encryption keys over an insecure channel, frame their transmission data, and maintain a protected chat stream without a central database or server intermediate.

## 📁 Repository Structure

The development of this network stack is broken down into two distinct engineering checkpoints:

* **[CHECKPOINT 1](./CHECKPOINT1.md) — Baseline Foundations:** Establishes the asynchronous duplex P2P socket routing architecture.
* **[CHECKPOINT 2](./CHECKPOINT2.md) — Secure Pipeline:** Integrates custom delimiter framing, a Diffie-Hellman dynamic key exchange, and AES-GCM 256-bit encryption.

## 🚀 Quick Start & Execution

Since this is a decentralized peer-to-peer system, both nodes run the exact same script (`server1.py`). The application resolves roles (Initiator vs. Follower) dynamically at runtime.

### Prerequisite Dependencies
Make sure you have the Python `cryptography` and `sympy` packages installed:
```bash
pip install cryptography sympy

Running the Application Locally

Open two terminal windows side-by-side to simulate two network peers (Jon and Jom):
Terminal 1 (Jon)
Bash

python3 server1.py
Enter your name: Jon
Who do you want to text: Jom

Terminal 2 (Jom)
Bash

Once both terminals cross-connect, they will automatically run a cryptographic handshake and initialize a fully secure, end-to-end encrypted tunnel.
🔑 Core Technical Specifications

    Architecture: Symmetric P2P Duplex Socket Mesh (No Central Server).

    Network Transport: TCP Stream (socket.SOCK_STREAM) bound locally on dual ports (50000 / 51000).

    Key Exchange: Automated Diffie-Hellman Handshake utilizing prime ranges up to 10,000,000 generated by sympy.randprime.

    Symmetric Encryption: AES-GCM (Authenticated Encryption with Associated Data) via the cryptography primitives layer, utilizing uniquely randomized 12-byte nonces for every transmission block.

    Stream Framing: Byte-level custom Delimiter-Based Processing.
